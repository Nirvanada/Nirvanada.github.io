Tensorflow

发表于 2016-07-02   |   分类于 Deep Learning

前言

机器学习在近些年受到了相关领域发展的快速推动，比如高性能计算、更高容量的存储和互联网技术。我们日常生活中的一些事情也或多或少的受到了机器学习和人工智能的影响，例如移动端的语音语义识别、图像分类和垃圾邮件检测等等，而这些应用在几十年前我们可能只在科幻小说里听说过，并且机器学习在股票市场模型或医疗模型中的应用已经极大程度的改变了我们的社会。不仅如此，对于车辆的自动导航，无人驾驶和机器人的相关领域，机器学习作为一种有效的工具也会在未来给社会带来更大的改变。

深度学习领域作为机器学习中一种，自从2006年以来已经呈现井喷式的发展。硅谷中的许多创业公司以及许多大型科技公司（Google，Facebook，Microsoft，IBM等）也在这一领域上有了不小的动作。而这一领域的发展也同时受到许多其他行业的高度关注，如New York Times，Bloomberg，BBC等。因此，许多学生和开源爱好者也都积极的向这一领域靠拢。目前为止许多深度学习的packages都已经开源，由Google团队开发的TensorFlow则是其中一个重量级、比较有代表性的library。

TensorFlow简介

TensorFlow是一个采用数据流图（Data Flow Graphs），用于数值计算的开源软件库。节点（Nodes）在图中表示数学操作，图中的线（Edges）则表示在节点间相互联系的多维数据数组，即张量（tensor）。它灵活的架构让你可以在多种平台上展开计算，例如台式计算机中的一个或多个CPU（或GPU），服务器，移动设备等等。TensorFlow 最初由Google Brain Team开发，用于机器学习和深度神经网络方面的研究，但这个系统的通用性使其也可广泛用于其他计算领域。

Tensorflow特点

灵活性

TensorFlow 不是一个严格的“神经网络”库。只要你可以将你的计算表示为一个数据流图，你就可以使用Tensorflow。你来构建图，描写驱动计算的内部循环。我们提供了有用的工具来帮助你组装“子图”（常用于神经网络），当然用户也可以自己在Tensorflow基础上写自己的“上层库”。定义顺手好用的新复合操作和写一个python函数一样容易，而且也不用担心性能损耗。当然万一你发现找不到想要的底层数据操作，你也可以自己写一点c++代码来丰富底层的操作。

可移植性

Tensorflow可以在CPU和GPU上运行，并可以运行在台式机、服务器、移动设备等等。笔记本上跑，没问题；在多个CPU上规模化运算，没问题；将你训练好的模型作为产品的一部分用到手机app里？没问题；将模型作为云端服务运行在自己的服务器上，或者运行在Docker容器里？都没问题。

多语言支持

Tensorflow有一个合理的c++使用界面，也有一个易用的python使用界面来构建和执行你的graphs。你可以直接写python或c++程序，也可以用交互式的ipython界面来用Tensorflow尝试些想法，它可以帮你将笔记、代码、可视化等有条理地归置好。当然这仅仅是个起点——你可以创造自己最喜欢的语言界面，比如Go，Java，Lua，Javascript，或者是R。

线性回归模型

线性回归可以定量描述变量间的关系，且被广泛的运用在各类场景中。模型的构成非常简单，即 $y = W * x + b$，首先创建x和y向量

接下来我们要利用 $x$ 和 $y$ 通过指定的模型训练方法来获取 $W$ 和 $b$，以使得利用模型得到的 $y$ 的预测值与真实值$y$足够接近，因此我们定义一个损失函数（Cost-Function）来评价两者的吻合程度。对于线性回归而言，可以使用均方误差这一指标来衡量，目标就是使指标取得最小值，损失函数定义为 $loss = \frac{1}{N} \sum{}{} (y_{pred} - y_{truth})^2$。在TensorFlow中，依然通过变量和算子来实现。

TensorFlow提供了梯度下降方法，即每次迭代使损失函数朝着其负梯度的方向逐渐下降，最终落到极小值点，并获取当前时刻的 $W$ 和 $b$，这里0.5表示为学习率（梯度下降速率）

利用TensorFlow进行聚类分析

线性回归是一种监督学习方法，也就是利用目标变量的值或标签去训练一个拟合这些数据的模型。但是在某些时候我们根本不知道目标变量是什么，因此这时候就需要利用非监督方法对数据进行进一步的了解，如聚类分析。这种方法也经常被用来进行数据的初步筛选。
